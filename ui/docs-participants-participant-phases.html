Participant phases (O3)

What labels and animation should integrators show for auditor/protocol/tool/line maturity — without treating hints as payout gates?

Participant phases and line phases are off-chain presentation tags on the Bob card — computed in cell/indexer/, not on-chain enums in the settlement cell.

Rule: phases alarm and explain; only L0/L1 tool paths punish (Mechanism).

Bob pipeline: Bob card · maturity: Maturity.

Two tag families

Participant phases (deriveBobCard.mjs)

Values: `unproven` · `alive` · `declining` · `detrimental`

EntityDriver
AuditorderiveAuditorPhase() — successful / failed / found, timeout streak
ProtocolderiveProtocolPhase() — submissions, exploits, claims
ToolderiveToolPhase() — successfulUses / failedUses, canonical flag

Line phase (this audit card)

Values: `nascent` · `steady` · `corroding` · `escalating`

deriveLinePhase() — maturityBand, trust legs, hint FAILs (wash-triangle, defended-spec-on-wash-line, dissolve tools). Hints: ../../meta/hint-registry.md.

Assignment signals (protocolRejectCount, dyad stats) feed sequence hints — Assignment (CRD).
