Structural upgrades

The shared toolkit is missing a check class — how does someone prove the gap, ship a fix, get jury adopt, and optionally mint UPG — without settling unrelated rows?

Story context

  • Book ch 10 · Issuance & positive blocks
  • Book ch 12 · Overlays

Structural upgrade adoption (UPG) is the governed path when the network’s canonical harness fails on a target O and the fix is a new registered primitive F — not a debate on one protocol’s declared spec. The network learns (Probationary → Official canonical tier) without rewriting the immutable settlement core.

Per-audit fail outside S on one row → Spec-gap overlay. One protocol’s app bug → ordinary audit / Gate C — not a network gap.

Module: StructuralUpgradeModule.sol · UPG mint via IssuanceModule.setStructuralModule (Where mint splits).

Escrow tranches

After fix `InBlock`, proposer may claim pass tranche during probation:

passTarget = upgradeProposalPayoutTarget × passPayoutBps / 10_000 (default 40%)

On `adoptStructuralUpgrade`: adopt tranche + UPG mint. Draws capped at upgradeClaimCapBps (default 5.4%) of general escrow via payStructuralUpgradeEscrow.

Jury credibility

Votes feed headcount, seniority-weighted, and credibility-weighted tallies (juryOkWC / juryNotOkWC). Adopt requires primary gates plus WC gates once data exists. Post-adopt: OK jurors call confirmJuryCredit for future modifier.
