Quickstart

Tool author

Register a checker so the whole network can use it. The growing toolkit is the network's real product — you build it.

1

Build a checker

A deterministic tool that runs against a contract's bytecode and produces an AUDIT_RESULT_V1 fingerprint. Two honest runs on the same code must give the exact same result.

2

Compute its tool id

Hash the tool's identity — name, version, build artifact, entrypoint — into its AUDIT_TOOL_V1 id. That id is how protocols name it and auditors run it.

3

Register it

Submit the tool id on-chain — as a verdict tool (the default), or a spec-validation tool for Gate A. One transaction, no stake, permissionless.

4

It earns canonical standing

Every audit that uses your tool and survives the challenge period builds its track record. Tools earn trust by surviving reality — not by a vote.

Your stake in the outcome

A weak tool loses trust

If your tool passes a contract that's later proven exploitable, that failure is recorded against the tool. A checker is only as valuable as its track record — reproducibly wrong is still wrong.
