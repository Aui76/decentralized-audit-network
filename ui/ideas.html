How ideas become real

Nothing here is true just because someone says so. This page explains, in plain terms, how a rough idea or a worry turns into a real, checkable fact on the network.

The one rule

An idea is "real" only when a stranger can re-run it and get the same answer — with that result recorded on-chain. Talk alone never ships. If it can't be reproduced, it doesn't count.

From talk to real, in three steps

  1. Someone says it. A worry, an idea, a "this looks wrong." Free, and proves nothing on its own — that's where almost everything starts.
  2. Someone does the work. They turn it into something anyone can re-run — a specific check on a specific contract — and back it with a deposit, so they have skin in the game.
  3. The chain records it. The result is posted on-chain, where anyone can run the same check and get the same answer. Now it's real — nobody has to take your word for it.

The whole philosophy in one line: opinions are cheap, reproduction is the currency.

For builders & operators — the full machinery

Ideas climb a ladder: talk → proposal → reproducible manifest → registered flaw-ID → on-chain fact → canon. Changes travel one of three channels — meta (repo / tools / membrane, via proposal → apply), settlement (claim / gap / exploit, stake + replay on chain), or interpretation (risk routing, Bob, the concern router — never pays or slashes).
