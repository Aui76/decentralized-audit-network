One ledger, many contracts

Chapters 1–3 named the grievance, the promise, and the product shape.

Every audited contract is a line on a public ledger. For each one the chain remembers roughly two things:

FieldMeaning
Value securedDeclared magnitude at submit — thermometer, not payout formula
Audit standingBinary — check still good, or failed because exploit reproduced

Add up value on lines that still stand → network TVL in the story sense: value the network is currently vouching for through unbroken audits.

Token supply grows only when the mechanism records real output — not when someone prints a narrative.

Skin in the game at every layer

RoleConsequence aligned with accuracy
AuditorPaid only after pass survives challenge window; reproducible miss → failed record
ProtocolBounty locked; valid challenge cannot be vetoed by politics
Bug-hunterPaid from reserves on reproduced miss — not fresh mint
Token holderNetwork valuable when it prevents and catches bugs

The ledger updates when tools and windows produce binary outcomes — not when a committee interprets intent.

Supply follows survived truth

New tokens enter only through mechanism output — chiefly confirmed success-path (positive-path) audits and tool standing earned in public. Discoverer payouts come from reserves already set aside.

Market price is outside the contracts. What the chain fixes is when minting is allowed at all.
