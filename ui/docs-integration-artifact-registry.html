Artifact registry

How do indexers QA codehash dedup** — and what are invariants R1–R5?

Story context

  • Book ch 9 · The audit cycle
  • Book ch 17 · Binary by choice
  • Book ch 19 · Gaming & defenses

Indexer QA for codehash-pin dedup on `AuditCell`. Two linked structures per ordinary submit must stay consistent — registry alone or audit row alone can lie if they drift.

On-chain (C3/C5): artifactRegistered, artifactToAuditId, audits[id].artifactHash on `AuditCell`.

Off-chain validator: cell/indexer/validateArtifactRegistry.mjs.

UI display rules: Protocol UI spec (staleness check).

Two structures

StructureKeyMeaning
Audit recordauditIdFull lifecycle row (state, deployedAddress, artifactHash, …)
Artifact registryartifactHashdeployedAddress.codehash at submit — global dedup lock

Invariants (R1–R5)

R1 — Active audit owns the lock

If auditStateOf(id) is not Invalidated, for hash = audits(id).artifactHash:

  • artifactRegistered[hash] == true
  • artifactToAuditId[hash] == id

Lock-holding states: Submitted, Assigned, InAudit, AwaitingWindow, Audited, InBlock, Claimed, Exploited.

R2 — Invalidated releases the lock

If state is `Invalidated`, for that row's hash:

  • artifactRegistered[hash] == false
  • artifactToAuditId[hash] == 0

Clear paths: spec challenge finalize / arbiter FAIL, integrity review FAIL, integrity void helpers.

R3 — Registry pointer matches row

If artifactRegistered[hash] == true, let mapped = artifactToAuditId[hash]:

  • auditExists(mapped)
  • audits(mapped).artifactHash == hash

R4 — Submit pins hash at call time

artifactHash is `deployedAddress.codehash` at `submitAudit` — not live extcodehash later. Proxy upgrades change live code without updating the row; integrators run a staleness check (Protocol UI spec).

R5 — Supersedes ≠ dedup bypass

supersedesAuditId is navigation only. Re-submitting the same bytecode still reverts ArtifactAlreadyAudited(existingId).

What the registry is / is not

Registry answersRegistry does not answer
“Is this bytecode already in an audit pipeline?”“Is this protocol safe?”
“Which auditId holds the dedup lock?”“Does this address match live code?” (staleness)
“Can the same code be submitted again?”PASS / CLEAN / composable (audit state + Bob)

After `Invalidated`, same bytecode may be submitted as a new auditId. After `Exploited`, the lock remains — new work requires new bytecode or invalidation on an open line.

Off-chain validator (O9)

import { validateArtifactRegistryConsistency } from "./validateArtifactRegistry.mjs";

const { ok, errors } = validateArtifactRegistryConsistency({
  auditId,
  auditState,           // name or 6 for Invalidated
  artifactHash,
  artifactRegistered,
  artifactToAuditId,
  mappedArtifactHash,   // optional R3 cross-check
});

Indexer workflow

  1. On AuditSubmitted, cache (auditId → artifactHash, deployedAddress) and registry from artifactRegistered / artifactToAuditId.
  2. On spec invalidation or integrity void → clear registry cache for that hash.
  3. On Exploited → keep registry cache (lock stays).
  4. Periodic reconcile: run validateArtifactRegistryConsistency per indexed auditId.
  5. Before “submit new audit” for bytecode H: if artifactRegistered[H], link UI to artifactToAuditId[H].
