Reading cell state

Primary state lives on `AuditCell` and linked modules (ClaimDisputeModule, CellEscrow, …). All audit rows are public; prefer stable named views over raw tuple decode when the cell exposes them.

Lifecycle tracking: subscribe to events per audit id — AuditSubmitted → assignment → accept → verdict → confirm. Fork branches: VulnerabilityClaimed, spec challenge events, integrity overlay (when wired).

On-chain vs off-chain: the cell stores commitments (resultRoot, specErrorsRoot, *RunDigest), not tool output. Verification = re-run declared tool against pinned bytecode and match digest (declare + re-run). See Wallets, re-run & verifier seam.
