Protocol quickstart {#protocol-quickstart}

The shortest path to post an audit and get a reproducible verdict.

  1. Have a tool. Registered spec-validation + verdict tool, or registerTool(toolId, isSpecTool).
  2. Submit. submitAudit(…) pins codehash, declares Gate A, locks bounty in escrow.
  3. Accept the auditor. protocolAcceptAuditor(id) or silence → auto-accept.
  4. Wait. PASS → challenge window → confirmAudit(id) → positive block.
  5. If challenged. Reproduction settlement — Vulnerability disclosure.

Protocol integration (detail)

Ordinary audit submit pins deployedAddress codehash, posts bounty to escrow, declares Gate A (specToolId, specErrorsRoot). Re-submitting identical bytecode reverts (ArtifactAlreadyAudited). Registry rules: Artifact registry.

Protocol accepts/rejects assigned auditor within decision window (ordinary audits). Fix audits skip protocol reject gate.

Auditor quickstart {#auditor-quickstart}

  1. Register — register(); hold = (position − 1) × 1 AUDIT on live 84532 (increment locked); first auditor: 0.
  2. Get assigned — CRD / FIFO assignment.
  3. Accept + re-run spec — acceptAudit(id, specErrorsRoot).
  4. Verdict — provePass / proveFail.
  5. Collect — survive window; confirmAudit(id).
