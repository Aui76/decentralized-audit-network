Five roles operate the network. Each acts in its own interest; incentives align toward reproducible security — not toward a shared committee.

Assignment policy: Assignment (CRD) · break path: Vulnerability disclosure.

Not here: pool draw mechanics, dyad cap, reject cap — Assignment (CRD). Bob phase tags and animation — Participant phases (O3).

Protocol

Posts a contract and funds a bounty, naming the check that applies. Accepts the assigned auditor, rejects within the decision window, or stays silent (→ auto-accept).

*Reject chooses who audits* — never whether the spec or verdict is wrong.

RiskBounty held during audit; surfaced bug recorded against the contract in the open
GainReproducible, timestamped record that the contract passed a specific check

Integrator surface: § Protocol — integrator surface below.

Auditor

Joins the pool, is assigned, runs the named tool, submits binary verdict, waits out the challenge period.

RiskPaid only at confirmation; proven miss → audit failed, reward never lands; record permanent
GainConfirmed pass earns mechanism reward; strong record raises future earnings
EntryPermissionless register(). Hold: (position − 1) × increment (live 84532: increment = 1 AUDIT locked; first registrar: 0)

Repeated silence → rotated out; re-registration open.

Tool author

Registers a deterministic checker — verdict tool or spec-validation tool for Gate A.

RiskTool that passes later-exploited contract → failedUses on-chain
GainAt canonicalThreshold uses (default 10), auto-canonicalize + CAN mint to proposer — verifiable count, no admin discretion

FMEA / class registry: ../../meta/fmea.md.

Bug-hunter

During the challenge period, proves a missed bug with a registered tool. Claim is proof, not allegation.

RiskStake forfeited if claim fails to reproduce
GainIndependent re-check reproduces bug → payment from reserves
EligibilityRegistered auditor; not protocol; not original auditor (post-PASS path)

Procedure: Vulnerability disclosure.

Observer

Any party — no wallet required — re-runs the check on live code and compares to the recorded result. Reproduction is what makes a verdict proof, not marketing.

Protocol — integrator surface

Protocol posts contracts, locks bounty, declares Gate A spec, and accepts or rejects assigned auditors within bounded windows.

Reject = who audits. Wrong spec → Gate A. Wrong PASS on code → Gate C.
