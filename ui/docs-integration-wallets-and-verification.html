Option F — the verifier seam (C10)

An optional cryptographic verifier that would harden the witness binding into an unforgeable proof. It is a stub today (IRunProofVerifier, the C10 seam) — not wired into any settlement path.

What it would harden

What is not yet cryptographic is the binding between an on-chain fingerprint and a genuine tool run. Today that binding rests on:

  • an independent re-runner who actually executes the tool, and
  • a 14-day window in which anyone can reproduce the run and challenge it.

The assumption is honest-majority over an open population given time to look — not trust in a single actor at a single moment.

What Option F is

A verifier — zero-knowledge or trusted-execution — that proves on-chain the statement "replay witness W on contract O → evaluator → FAIL," replacing the re-runner-plus-window with an unforgeable proof of the same fact. It proves exactly what the Fingerprint Technical Note already encodes; it changes nothing else.

The cell is signerless and hash-anchored, so the verifier attaches as a future seam without changing the encoding. Build when the goal is to remove the honest-majority + window assumption; not a necessary condition for pay or withhold.

Wallet integration

  1. Compute digest via eip712Digests.mjs or cathedral on-chain views (must agree).
  2. Present eth_signTypedData_v4 using buildClaimTypedData / buildVerdictTypedData.
  3. Indexers may store both runDigest (Bob re-run binding) and optional eip712Digest for audit trails.
