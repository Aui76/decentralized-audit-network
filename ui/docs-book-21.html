One row, not a brand badge

The network certifies one pinned audit case per audit id: deployment address, bytecode hash at submit, spec binding at the door (Gate A).

“Acme Protocol ✅ audited” without the row is marketing. Honest copy: “Audit #47 on 0xBBB — PASS finalized — for bytecode pinned at submit.”

Supersedes links = navigation only — no inherited pass across deployments.

Proxy upgrades: compare live bytecode to pin; show staleness when they diverge.

Timeouts = liveness — rotate or expire; not automatic guilt.

What the protocol sees

Per-row machine: prepare → submitted → assigned → in audit → awaiting window → in block → (claimed → exploited if miss reproduces).

Reject picks who, not truth. Wrong spec → Gate A. Wrong pass on code → Gate C (ch. 11).

What the auditor sees

Registered → assigned → in audit → binary verdict → window → paid path (confirm) or lost path (exploited).

FAIL at Gate B still enters claimed with auditor as claimant — same break spine.

Pool silence → reassignment; register again open — liveness, not guilt verdict.

What the discoverer sees

Registered auditor, not protocol, not original auditor (ch. 19).

Investigate → file claim + claim stake → await re-run → win ( reserves + boost) or loss (stake slashed).

What everyone else sees

Observers — no wallet; re-run tool; compare `resultRoot`.

Tool authors — register; accumulate uses; canonization path (ch. 13).

Participant phases — off-chain paint (ch. 16); never replace the seven states.

No PDF thread

No draft report loop, no re-grade call. At most one primary action — or wait with a visible clock.

Chapter 23 names the membrane built on these machines.
