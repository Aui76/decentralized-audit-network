Bugs & disputes

A pass isn't paid right away. The challenge period is the window where a missed bug can be claimed — and settled by re-checking the original contract.

Filing a claim

During the challenge period, anyone in the pool — except the contract's protocol or its original auditor — can prove a bug the auditor missed and file a claim. Filing requires a stake. The claim is a proof, not an accusation: it pauses the audit and starts the resolution.

The independent re-check

The protocol can open a dispute, which it funds. An independent auditor — not the protocol, the original auditor, or the claimant — re-checks the original contract, re-running the claimed tool on the same deployed code.

How it settles

If the re-check reproduces the bug, the original audit is marked exploited and the finder is paid from reserves — never from new supply. If it can't be reproduced, the claim is rejected, the finder loses their stake, and the original audit stands.

If nothing happens

If a claim is never taken up and its window runs out, it's closed: the finder's stake is slashed, no one is paid, and the original audit is restored. A claim you can't back costs you.

Always on the original — never a patch

Settlement is always the result reproduced on the original contract. Patching the contract, or finishing any other procedure, never settles a claim for you. The network acts only on the same code, the same tool, the same result.

A fail at the check — the auditor's own verdict — opens this same resolution. Either way, the question is one thing: does the bug reproduce on the contract that was audited?
