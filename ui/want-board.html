Free · no transaction · a wishlist, not a verdict

What should we check next?

A public wishlist for getting smart contracts security-checked. Worried about a contract? Point at it and say "someone should check this" — free, one click, no wallet transaction. The board floats the contracts most worth checking to the top — judged mostly by how much money is at risk — and your click is a nudge, not the decision. When a contract rises high enough, someone funds a real audit.

Add a contract to the board

Must be live on-chain — that's the only thing that can be reproduced later.

Your click is a nudge, not a verdict. The ranking is driven mostly by the facts in each row — money at risk, who's exposed, whether it's been checked before. Adding a "want" is free and just says "I'd like this looked at"; it proves nothing on its own. A real check only happens when someone funds an audit on Post a bounty.

* "Resembles a known failure" (the red tag) means this contract's code looks similar to code that has been hacked or has failed before — the network keeps an on-chain memory of past bug types and flags matches. It's a heads-up, not a verdict: a reason to look, not proof anything is wrong. (Could also read as "looks like code that's failed before" or "similar to a past exploit.")

† What's on the board today: these are contracts the network has already audited (passed) or is currently auditing (live) — the list is built from the network's own audit records. A genuinely unaudited contract only appears when someone adds one above ("Add a contract"). So until the board is fed a source of un-checked deployments, "Unaudited only" will be nearly empty — a known gap between what this page is for and where its data comes from, not a bug.
