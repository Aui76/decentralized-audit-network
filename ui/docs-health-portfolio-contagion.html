Portfolio contagion

When does a confirmed exploit imply other InBlock audits are exposed** — and how does Bob dampen (not settle) the card?

Story context

  • Book ch 19 · Gaming & defenses
  • Book ch 22 · Reading the chain

Off-chain SIR-style model over InBlock audits that share FMEA spec-gap overlap with confirmed exploits. No on-chain contract. Indexer-only; optional Bob fuse when portfolioContext is supplied.

Module: `cell/indexer/deriveContagion.mjs` (O5 organ).

Flow vs stock

A portfolio can look flow-healthy while a vulnerability class is outbreaking (R₀ > 1). Integrators need both; neither alone means "safe."

Contagion reads `toolKnownGaps` from `FmeaRegistry` (claim exploits) + `SpecGapModule` (spec-gap adopt/concede) — merged at indexer tick. O4 scope symbols are separate.

Exposure rule

An InBlock audit is exposed to class C when:

  1. Its specToolId lists C in toolKnownGaps, and
  2. C has a recent exploit on a different audit (same class, spec-gap overlap — not mere shared tool id), and
  3. The audit is not the exploit source row itself.

Exposure is class-local, not "same tool as victim."

R₀ per class

For each class with recent exploits:

R₀ = (exploitCount × exposureCount) / reauditRate
R₀LabelMeaning
≤ 1containedRe-audits / gap closure keep pace
> 1outbreakExposed InBlock rows accumulate faster than re-audits

portfolio.outbreakClasses lists classes with R₀ > 1. portfolio.networkR0 is the max across outbreak (else max contained).

Bob fuse

When deriveBobCard receives portfolioContext, it runs derivePortfolioContagion then `applyContagionToBobCard`:

ConditionCard effect
Outbreak exposure on this auditCLEAN → WEAK; INVEST → WAIT; confidence capped LOW; reviewTier → watch; pattern portfolio-fuse
Advisory only (reauditAdvisory, no outbreak)Softer dampers; implication notes overlap
Missing portfolioContextintegratorWarnings: MISSING_PORTFOLIO_CONTEXT — fuse skipped

R₀ > 1 means priority re-audit candidate for overlapping class — not global mint denial, auto-slash, or "contract vulnerable."

Flow-side hints (card.flow) remain separate — Network flow.
