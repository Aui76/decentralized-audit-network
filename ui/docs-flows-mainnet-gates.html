Mainnet gates

What must be true before removing the demo / testnet-grade label and treating the network as mainnet-ready for real value?

Story context

  • Book ch 20 · What we defer

Mainnet gates are the checklist for when a testnet-grade deployment may carry real economic weight — not when the mechanism first becomes reproducible. Base Sepolia today already runs the full settlement railroad (cell, overlays, confirm, claim, dispute); mainnet is the same bytecode with production parameters, hardened organs, and issuance signals enforced at confirm.

One mechanism rule: cell bytecode is identical testnet ↔ mainnet; divergence is parameters and posture only — not a second settlement story (What we defer (story)).

Integrator rule

Posture (demo label)

The demo / testnet-grade label marks deployment posture — compressed windows, exercise-driven log, no production-scale bounties — not whether fail-on-pin pays from escrow on Sepolia.

Remove the label only when launch gates and issuance gates evidence justify real value — not at first deploy.
