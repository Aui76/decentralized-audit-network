Appendix A — Module wiring

Which `setDisputeModule` index maps to which overlay — and what is the minimal wire + test surface for integrators?

Module wiring is how L1 satellites attach to `AuditCell`: dispute module slots, one-time wire() calls, and the Foundry cell tests that oracle each overlay path.

Procedure specs: Spec arbiter · Spec-gap overlay · Integrity review · Structural upgrades · Vulnerability disclosure.

Integration hub: Integration.

Dispute module indices

AuditCell.setDisputeModule(index, moduleAddress) — settlement mutators call the wired module at each index:

IndexModuleProcedure spec
0Claim (break path)Vulnerability disclosure
1Spec-gapSpec-gap overlay
2Spec arbiterSpec arbiter
3Integrity reviewIntegrity review

Structural upgrades use a separate seam — IssuanceModule.setStructuralModule — not a dispute index (Structural upgrades).

Wiring snippets

Spec arbiter (index 2)

cell.setDisputeModule(2, address(specArbiterModule));

Integrity review (index 3)

cell.setDisputeModule(3, address(integrityReviewModule));
integrityReviewModule.wire(address(cell), address(specArbiterModule));

Spec-gap (dedicated setter)

cell.setSpecGapModule(module);
module.wire(cell);

Structural upgrades (issuance seam)

IssuanceModule.setStructuralModule(address(structuralUpgradeModule));
