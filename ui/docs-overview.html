Overview

What the Audit Cell is, and why it exists.

The problem

A smart contract can't be patched. Once it's deployed, the code is permanent — if it's wrong, the loss is permanent too. So getting it right before deployment isn't a quality goal; it's survival.

Today the usual answer is the professional audit: a protocol pays a firm, waits weeks, and gets a report. That has three built-in flaws:

  • The auditor is paid by the audited. The incentive bends toward verdicts that don't create friction.
  • No stake after the report. If a missed bug is exploited later, the fee isn't returned.
  • The result isn't reproducible. "Audited by X" is an opinion you have to trust — not a result you can recompute.

The idea

The Audit Cell replaces the opinion with a result anyone can reproduce. A contract is checked by a named tool against a written spec. The verdict is binary — pass or fail — and anyone can run the same tool on the same code and get the same answer. No firm, no committee, no discretion.

What makes it different

  • Binary. Pass or fail. Nothing in between.
  • Reproducible. Anyone can recompute the result from the live code.
  • Open. Anyone who qualifies can take part. No gatekeeper.
  • Stake that lasts. The auditor is paid only after the contract survives a challenge period — not at the moment of the verdict.
  • Self-funding. The reward comes from the mechanism itself, not from the protocol's goodwill.
  • Accumulating. Every audit adds to a shared, growing toolkit of checks.

The two outcomes

Pass. The contract clears its check and survives the challenge period. The auditor is paid, and the toolkit grows.

A bug is found. An independent auditor re-checks the original contract. If the bug is reproduced, the audit is marked failed and the finder is paid from reserves — never from new supply. If it can't be reproduced, the claim is rejected. Failures are recorded, never rewarded with fresh tokens.

The real product

The point isn't any single verdict. It's the toolkit — a growing, reusable set of machine-checkable checks that anyone can run. Individual audits are how it grows.
