One audit, one block

Block size is one: each confirmed audit is its own positive block.

Mint pace follows activity — EMA, LP cap, manipulation/down signals. Policy knobs, not discretionary payouts.

Each block splits: auditor reward, treasury (LP + escrow slices → reserves), founder accrual — hard-capped.

Proof-of-audit

Supply grows only when the network demonstrably did its job — chiefly survived audits on the success path (positive path).

The application-level block chain records POS and CAN. Break-path resolutions update state and redirect escrow — they do not mint for the miss itself.
