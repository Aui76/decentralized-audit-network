The reframe

Chapters 1 and 2 named the problem and the promise: replace trust in a name with a reproducible verdict. That promise could still be read as “another audit market.”

It is not. The primary product is not audit verdicts. It is a canonical, growing set of verification tools — the toolkit.

Individual audits are how the toolkit grows. The audits are exhaust; the toolkit is the engine.

What every audit must leave behind

Each submission posts three things:

ArtifactMeaning
ProofWhat was run and what came out — inspectable, not a narrative
Registered toolsOn-chain IDs, not “best practice”
Optional new toolContribution the next team can cite unchanged

The next audit starts from whatever already exists plus whatever this one adds. The set grows monotonically.

Why this framing lasts

PropertyMeaning
Tools are reproducibleSame tool + bytecode → same answer; pass is computation with provenance
Complexity ramps graduallySimple binary checks first; hard problems when the toolkit can state them
Incumbents can adoptCanonical rail both sides run — human judgment on top, same base underneath
Specs sharpen with toolsBetter tools → sharper declarations at submit

How a tool earns standing

No committee vote. No brand deal. Reputation accumulates through use:

  1. Anyone can register a deterministic checker — permissionless.
  2. An auditor cites it; the audit runs its challenge window.
  3. Clean confirm → successful-use count increments on-chain.
  4. Threshold crossed → canonized; author earns CAN mint.
  5. Later exploit → failure count increments — public signal the checker is suspect.

Every step is observable and binary. Tool standing is what happened when people bet real bounties on this checker.

What this means for everyone else

RoleMeaning
ProtocolPins launch to checks others can cite next quarter
AuditorExecutes registered tools; posts proof
Tool authorShips reusable checkers; upside is canonization
StrangerCompares recorded tools to canonical set; re-runs if they choose

The loop, restated

Token flows, pool rules, and escrow mechanics serve this loop: tools get registered, cited, tested under fire, and canonized or flagged. Audits are the stress test; the toolkit is what survives.
