Audit mutex

If two fights could apply to the same audit row, which one is legal now — and what must wait?

Story context

  • Book ch 11 · The break path
  • Book ch 12 · Overlays
  • Book ch 19 · Gaming & defenses

Each ordinary `auditId` runs at most one dispute lane at a time. The cell exposes the active lane in `disputeLaneOf(auditId)` — the on-chain lock that blocks competing openers.

Related: Vulnerability disclosure (Claim lane) · Spec arbiter (Spec lane) · Integrity review (Integrity lane) · Audit lifecycle.

One row, one lane

auditId N  ──►  disputeLaneOf  ∈  { None, Spec, Integrity, Claim }
                      │
                      └── first opener acquires; second reverts

First acquire wins. A second challengeSpecInvalid, openIntegrityReview, or claimVulnerability / proveFail while another lane is active reverts.

Route the dominant question first. Mixing lanes on one row is how participants argue past each other while nothing settles.

Wrong-spec → Spec (Gate A). Missed bug → Claim (Gate C).

Precedence (what waits)

When a lane is active, the cell blocks conflicting moves on the same `auditId`:

PriorityIf active…Blocks (typical)
1Spec challengeconfirmAudit, claim, integrity-open, in-audit verdict
2Integrity reviewconfirmAudit, claim, spec-open
3Claim laneconfirmAudit, integrity-open, spec-open
4—Main-phase eligibility (assign, accept, prove, confirm)

Law vs signal

LayerScopeRole
Constraint (law)One auditId`disputeLaneOf` — only one lane open; enforces routing above
Amplifier (signal)Portfolio / BobHints suggest scrutiny — they do not open or close lanes by themselves

Triangle alarm — hint to scrutiny, not slash

Off-chain manipulation hints (wash-triangle, spec-defended-unverified) route scrutiny. They do not move money, open disputeLaneOf, or slash by themselves.

Heuristics have false positives. Auto-slash on hint → false-positive loss + weaponized griefing.

Pipeline

1. Hint fires (off-chain)     → Bob WALK / integrityReviewSuggested — advisory
2. Alarm escalates bar        → confirm/mint blocked until independent re-run path runs
3. Objective result           → slash / invalidate / clear only if tool proof contradicts line

Default damper for wash/collusion: Integrity review. Spec smells: Spec arbiter.

Health / signals: Signals · Bob: Bob card.

Integrator discipline

On-chain `InBlock` ≠ spec verified. Surface Bob card + spec arbiter status before “verified” copy. During open spec challenge, defend replay alone does not clear spec (P5d / spec-defended-unverified).

Legacy cleanup

If integrity void or spec invalidation lands while a stale `Claimed` row exists, the cell clears the unresolved claim before void.
