Assurance tiers

What do Standard / High badges mean** — and what do they not change on-chain?

Story context

  • Book ch 16 · Reputation signals

Standard and High are informational labels for cards and integrator UI — not settlement inputs (Settlement vs signal). Tiers never move pay, mint, assignment, verdict, or claim outcome.

Every audit gets the same challenge window and claim stake at submitAudit. The contract does not ask "Standard or High?" Tiers are badges an indexer may show. Build wallet flows from contract state only, not from tier badges.

What indexers derive

SignalMeaning for UI
Assurance tier (standard / high)Badge from declared tool count and optional snapshot metadata
Complexity phase (0–3)Launch ramp from lines of code or bounty USD (reference pricing)
Multiplier label (1×–20×)Protocol AUDIT balance at submit — copy only

Typical inputs

FieldMeaning
bountyWeiLocked bounty at submit
linesOfCodeArtifact metadata (optional)
declaredToolCountNon-spec tools declared at submit
protocolBalanceWeiProtocol balance for multiplier label
auditTokenPriceUsdUSD bands (reference default 0.10)

Optional snapshot fields (onChainAssuranceTier, multiplierAtSubmission) affect display only when present.

Typical output

{
  assuranceTier: "standard" | "high",
  complexityPhase: 0..3,
  multiplierLabel: "1x" | "2x" | ...,
  disclaimer: "informational only — ..."
}

Discoverer boost is separate — reputation history, not tier badges (The discoverer's reward).

Organ: cell/indexer/deriveTier.mjs · test: npm run test:organs-x3.

What tiers do not do

  • Do not scale discoverer payout, positive-block mint, or treasury split
  • Do not gate assignment, verdict, confirm, or witness settlement
  • Do not substitute for discoverer boost
