Claim-resolution scorecard

How do indexers score post-claim conduct** — without changing settlement?

Story context

  • Book ch 16 · Reputation signals
  • Book ch 19 · Gaming & defenses

Off-chain behavioral telemetry for post-claim dispute markets. Maps who funded the dispute × what reproduction on O showed per claim, then rolls up protocol seriousness bands.

Not on-chain truth — interprets cell events + audit rows. Does not change settlement, slash, or payout. Alarm and reputation input only (PK-2 wash-dyad flirt aligns here).

Per-claim row

Input: ClaimLifecycleSnapshot (from events + optional tx payer inference).

Core output fields:

FieldMeaning
disputeFunderprotocol · claimant · other · none
reproduceOutcomepass_on_O · fail_on_O · none
resolutionvindicated · exploited_repro · exploited_expire · procedural_reject · pending · legacy_fix_cycle
behavioralCellfunder:reproduce:resolution (e.g. protocol:pass_on_O:vindicated)
preVerdictWho carried before reproduction closed (decline / fund / stall signals)

API: deriveClaimScorecardRow(snapshot) · inferDisputeFunder(snap) · funderOutcomeCell(role, reproduce, kind).

Snapshot builder

buildClaimSnapshot(input) accepts flat indexer fields:

FieldMeaning
resolutionEventClaimVindicated · OriginalAuditExploited · ClaimExpired · ClaimRejectedProcedural · pending
disputeFunderRoleInferred or explicit funder role
disputeFunderAddress / disputeAuditIdDispute re-audit row
protocolDeclinedFundingProtocol declined to fund dispute (proposal B)
protocolAuditorCompletedOn-chain dyad counter — repeat-dyad / wash signal
proceduralRejectProcedural reject only (proposal A)

Golden snapshot: fixtures/claim-resolution-gate0-v3-r6.json (Gate0 R6 — protocol funded, PASS on O, claim vindicated).

Protocol aggregate

aggregateProtocolScorecard(rows, { protocol?, period? }) → counts, rates, funder×outcome matrix, wash-dyad flirt score, seriousness band:

BandMeaning (heuristic sketch)
seriousLow resolution gap, high repro rate, low expire-without-repro
wash_dyad_flirtHigh expire rate + elevated wash score
lazy_or_evadingHigh gap, low repro
mixedBetween
no_claimsEmpty cohort

Report: formatProtocolScorecardReport(agg) — CLI text for claimResolutionFleetLog.mjs report.
