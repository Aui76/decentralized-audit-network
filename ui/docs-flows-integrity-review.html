Integrity review

Should this audit row be independently re-run — and if the reviewer’s FAIL sustains, should the line void before or after confirm rules apply?

Story context

  • Book ch 12 · Overlays
  • Book ch 19 · Gaming & defenses

Integrity review is a paid, tool-gated re-check of the audit line on pinned contract O when process fraud or collusion is plausible enough to warrant independent scrutiny. Settlement follows binary tool replay (integrityRunDigest, AUDIT_INTEGRITY_RUN_V1) — not Bob hints alone.

Missed bugs within S → Vulnerability disclosure (Gate C). Bad declaration → Spec arbiter. Fail outside S → Spec-gap overlay. Lane lock: Audit mutex (Integrity lane).

Module: IntegrityReviewModule.sol · Off-chain hint: Bob integrityReviewSuggested — advisory only; a human/opener must call openIntegrityReview (Network flow).

Integrity escrow bucket: Escrow architecture.

Where it sits (Integrity lane)

Bob hint (optional) ──► openIntegrityReview(O) ──► Integrity lane locked
                              │
                              ├── submitIntegrityVerdict (PASS / FAIL)
                              ├── protocol contest (optional)
                              └── finalize / expire
                                        │
                    Cleared ──► line continues
                    Sustained ──► Invalidated (void rules per state)

Eligible at open: AwaitingWindow, InBlock (and Audited where applicable).

While `Open` or `VerdictSubmitted`: blocks confirmAudit and claimVulnerability on O. Spec challenge and Integrity review exclude each other (Mutex).

Wiring: Appendix A — Module wiring.

InBlock sustained FAIL (D-2)

No positive-block mint recall after InBlock — by design. Mint already landed in auditor free balance; on-chain recall is evadable.

Teeth that remain: pre-mint window still blocks confirm via lock; sustained FAIL increments `failed++`, clears artifact pin, and applies exclusion — recall-proof. Same rule testnet ↔ mainnet.

Integrators: treat sustained FAIL on an `InBlock` row as a failed line for reputation and pin, not as “undo the mint.”
